# 需求文档

## 1. 应用概述

### 1.1 应用名称
台风知识学习与测评系统

### 1.2 应用描述
一个支持电脑、手机、平板多端访问的网页应用,用户通过该系统学习台风形成相关知识,完成前测与后测,并可与AI对话代理进行交互。系统记录用户学习数据与测评结果。研究人员可通过后台管理页面查看统计数据、管理学习内容、配置系统参数。

## 2. 用户与使用场景

### 2.1 目标用户
- 普通学习者:需要学习台风知识并完成测评
- 研究人员:需要查看统计数据、管理学习内容、配置系统参数

### 2.2 核心使用场景
- 普通学习者登录系统后,按照「前测-知识学习-后测」的流程完成学习任务,系统记录全程数据并在完成后给予反馈
- 研究人员登录后台管理页面,查看用户测评数据与行为统计,管理学习内容,配置AI接口参数

## 3. 页面结构与功能说明

### 3.1 页面结构

```
├── 登录页
├── 学习主页(普通用户端)
│   ├── 左侧区域
│   │   ├── 目标提示区
│   │   └── 时间进度条与策略建议区
│   ├── 中间区域
│   │   └── 知识点讲解区
│   └── 右侧区域
│       └── AI对话代理区
└── 后台管理页面(研究人员端)
    ├── 侧边栏导航
    ├── 前后测统计页
    ├── 用户行为统计页
    ├── 进度管理页
    ├── 内容编辑页
    └── API配置页
```

### 3.2 登录页

#### 3.2.1 功能说明
- 用户输入用户名完成登录
- 用户名同步记录到数据库
- 登录成功后根据用户角色跳转:
  - 普通用户进入学习主页
  - 研究人员进入后台管理页面

### 3.3 学习主页(普通用户端)

#### 3.3.1 整体布局
- 采用响应式设计,适配电脑、手机、平板
- 电脑端:左侧两个分区、中间一个分区、右侧一个分区
- 移动端:分区垂直排列,保持功能完整性

#### 3.3.2 左侧区域

**目标提示区**
- 展示当前学习目标与任务说明

**时间进度条与策略建议区**
- 展示学习进度条
- 提供策略建议内容

#### 3.3.3 中间区域-知识点讲解区

**功能说明**
- 展示台风形成相关知识点内容
- 提供「开始学习」按钮
- 提供「完成学习」按钮

**开始学习流程**
- 用户点击「开始学习」按钮
- 弹出前测题界面
- 前测题为选择题形式
- 前测题示例:
  - 题目1:台风形成的主要条件是什么?
    - A. 高温高湿的海洋环境
    - B. 寒冷干燥的陆地环境
    - C. 高海拔山区环境
    - D. 沙漠环境
  - 题目2:台风的能量主要来源于?
    - A. 太阳辐射
    - B. 海洋热量
    - C. 地热能
    - D. 风能
  - 题目3:台风中心被称为?
    - A. 台风眼
    - B. 台风壁
    - C. 台风核
    - D. 台风心
- 用户答题过程中禁止进行网页其他操作
- 前测题答案记录到数据库
- 完成前测后展示知识点内容

**完成学习流程**
- 用户学习知识点后点击「完成学习」按钮
- 弹出后测题界面
- 后测题为选择题形式
- 后测题示例:
  - 题目1:台风形成需要海水温度达到多少度以上?
    - A. 20℃
    - B. 26.5℃
    - C. 30℃
    - D. 35℃
  - 题目2:台风按强度分为几个等级?
  - 题目3:台风登陆后强度会如何变化?
- 用户答题过程中禁止进行网页其他操作
- 后测题答案记录到数据库
- 完成后测后触发撒花庆祝效果

#### 3.3.4 右侧区域-AI对话代理区

**功能说明**
- 提供AI对话交互界面
- 用户可输入问题与AI进行对话
- 展示对话历史记录

### 3.4 后台管理页面(研究人员端)

#### 3.4.1 侧边栏导航

**功能说明**
- 展示导航菜单,包含以下选项:
  - 前后测统计
  - 用户行为统计
  - 进度管理
  - 内容编辑
  - API配置
- 提供「切换到普通用户端」入口

#### 3.4.2 前后测统计页

**功能说明**
- 展示所有用户的前测与后测得分数据
- 数据以列表形式呈现,包含:
  - 用户名
  - 前测得分
  - 后测得分
  - 测评完成时间

#### 3.4.3 用户行为统计页

**功能说明**
- 展示所有用户的操作行为记录
- 数据以列表形式呈现,包含:
  - 用户名
  - 操作类型(如:开始学习、完成前测、查看知识点、完成后测、AI对话等)
  - 操作时间

#### 3.4.4 进度管理页

**功能说明**
- 展示所有用户的学习进度状态
- 提供「重置进度」操作:
  - 研究人员选择指定用户
  - 点击「重置进度」按钮
  - 该用户的学习进度恢复为初始状态(未开始前测)

#### 3.4.5 内容编辑页

**功能说明**
- 展示当前台风知识讲解区的内容
- 提供编辑功能:
  - 研究人员可修改知识点文本内容
  - 点击「保存」按钮后更新内容
  - 更新后的内容在普通用户端实时生效

#### 3.4.6 API配置页

**功能说明**
- 展示当前AI对话代理的API配置信息
- 提供配置编辑功能:
  - 研究人员可修改API端点地址
  - 研究人员可修改API参数(如:模型名称、温度参数等)
  - 点击「保存」按钮后更新配置
  - 更新后的配置在AI对话代理中实时生效

## 4. 业务规则与逻辑

### 4.1 用户登录规则
- 每次进入网站必须输入用户名
- 用户名记录到数据库
- 系统根据用户角色判断跳转页面:
  - 普通用户跳转至学习主页
  - 研究人员跳转至后台管理页面

### 4.2 学习流程规则
- 必须按照「前测-知识学习-后测」顺序进行
- 点击「开始学习」触发前测
- 完成前测后才能查看知识点内容
- 点击「完成学习」触发后测
- 完成后测后触发撒花庆祝

### 4.3 测评规则
- 前测题与后测题均为选择题
- 做题期间禁止进行网页其他操作
- 所有答案记录到数据库

### 4.4 数据记录规则
- 用户名记录到数据库
- 前测题答案记录到数据库
- 后测题答案记录到数据库
- 用户操作行为记录到数据库

### 4.5 后台管理权限规则
- 仅具有管理员角色的研究人员可访问后台管理页面
- 普通用户无法访问后台管理页面

### 4.6 内容与配置更新规则
- 研究人员在内容编辑页修改知识点内容后,普通用户端实时生效
- 研究人员在API配置页修改配置后,AI对话代理实时生效

### 4.7 进度重置规则
- 研究人员重置指定用户进度后,该用户的学习状态恢复为初始状态
- 重置操作不影响已记录的历史数据(前后测答案、行为记录等)

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 用户未输入用户名尝试登录 | 提示用户输入用户名 |
| 普通用户尝试访问后台管理页面 | 拒绝访问,提示权限不足 |
| 用户未完成前测尝试查看知识点 | 限制访问,提示需完成前测 |
| 用户未学习知识点尝试点击完成学习 | 允许操作,触发后测 |
| 用户做题过程中尝试切换页面区域 | 禁止操作,保持测评界面 |
| 研究人员保存内容编辑时内容为空 | 提示不能保存空内容 |
| 研究人员保存API配置时端点地址格式错误 | 提示端点地址格式不正确 |
| 研究人员重置进度时未选择用户 | 提示需选择用户 |
| 数据库记录失败 | 提示用户稍后重试 |

## 6. 验收标准

### 6.1 普通用户端验收路径

1. 用户在登录页输入用户名,成功登录进入学习主页
2. 用户在知识点讲解区点击「开始学习」按钮,弹出前测题
3. 用户完成前测题,前测答案记录到数据库,展示知识点内容
4. 用户查看台风形成知识点内容后,点击「完成学习」按钮
5. 弹出后测题,用户完成后测题,后测答案记录到数据库
6. 完成后测后,网页展示撒花庆祝效果

### 6.2 研究人员端验收路径

1. 研究人员在登录页输入用户名,成功登录进入后台管理页面
2. 研究人员在侧边栏导航点击「前后测统计」,查看所有用户的测评得分数据
3. 研究人员在侧边栏导航点击「内容编辑」,修改台风知识讲解区内容并保存
4. 研究人员在侧边栏导航点击「进度管理」,选择指定用户并重置其学习进度
5. 研究人员在侧边栏导航点击「API配置」,修改AI对话代理的API端点和参数并保存
